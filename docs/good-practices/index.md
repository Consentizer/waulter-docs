# Good Practices

These guides cross-reference multiple Waulter features to show how they work together in real-world setups. Read these after you understand the individual features.

| Guide | Description |
|-------|-------------|
| [DEV & Testing Workflow](dev-testing.md) | How to safely develop and test Waulter before deploying to production |
| [Multilanguage Setup](multilanguage.md) | Setting up and managing consent banners in multiple languages |
| [Naming Conventions](naming-conventions.md) | Consistent naming for configurations, scenarios, and GTM elements |
| [Performance](performance.md) | How Waulter affects Core Web Vitals and how to minimise impact |
| [Styling & Brand Customization](styling.md) | Applying brand colours, testing contrast, CSS overrides |

## Who should read these?

- **Developers** implementing Waulter for the first time — start with [DEV & Testing](dev-testing.md)
- **Agency teams** managing multiple clients — focus on [Naming Conventions](naming-conventions.md) and [DEV & Testing](dev-testing.md)
- **Designers** customising the banner — see [Styling & Brand Customization](styling.md)
- **Performance engineers** — review [Performance](performance.md)
- **Multilingual sites** — follow the [Multilanguage Setup](multilanguage.md) guide
