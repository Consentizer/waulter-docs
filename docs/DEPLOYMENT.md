# Deployment — Azure Static Web Apps

This project uses two Azure SWA resources — **production** and **stage** — each
backed by a separate GitHub branch and deployment token.

| Environment | Branch        | Domain                    | Secret                                  | Resource Group          |
|-------------|---------------|---------------------------|-----------------------------------------|-------------------------|
| Production  | `main`        | https://docs.waulter.eu   | `AZURE_STATIC_WEB_APPS_API_TOKEN`       | `rg-production`         |
| Stage       | `development` | https://docs.stage.waulter.eu | `AZURE_STATIC_WEB_APPS_API_TOKEN_STAGE` | `rg-stage`              |
| PR preview  | PR branch     | Auto-generated by SWA     | (inherits from target branch)           | —                       |
| Local dev   | any           | http://localhost:8000     | —                                       | —                       |

## GitHub Secrets

Both tokens must be configured in the repository under
**Settings → Secrets and variables → Actions**:

| Secret Name                            | Source                        | Used by        |
|----------------------------------------|-------------------------------|----------------|
| `AZURE_STATIC_WEB_APPS_API_TOKEN`      | Production SWA → Manage token | `main` deploys |
| `AZURE_STATIC_WEB_APPS_API_TOKEN_STAGE`| Stage SWA → Manage token      | `development` deploys |

## One-time Azure setup — Production

1. **Create the SWA resource** in Azure Portal:
   - Resource group: `rg-production`
   - Plan type: Free tier is sufficient for documentation
   - Region: West Europe (closest to waulter.eu)
   - Deployment source: **GitHub** → authorise, select
     `Consentizer/waulter-docs`, branch `main`
   - Build preset: **Custom**
   - App location: `/`
   - Output location: `site`
   - Click **Review + Create**

2. **Copy the deployment token**:
   - After creation: SWA resource → Manage deployment token → Copy
   - In GitHub repo → Settings → Secrets and variables → Actions
   - Create secret: `AZURE_STATIC_WEB_APPS_API_TOKEN` → paste token

3. **Trigger first deploy**:
   - Push any commit to `main`, or manually run the workflow
   - The site will be live at the default SWA URL:
     `https://<auto-generated-name>.azurestaticapps.net`
   - Verify the site loads correctly before adding the custom domain

## Custom domain: docs.waulter.eu (Production)

### Step 1 — Add custom domain in Azure Portal

- Production SWA resource → Custom domains → Add
- Enter: `docs.waulter.eu`
- Azure will show you a **CNAME validation record** and the **CNAME target**
  (format: `<name>.azurestaticapps.net`)
- Copy both values before moving to Wedos

### Step 2 — Create DNS records in Wedos CZ

Log in to [client.wedos.com](https://client.wedos.com) → DNS Manager →
select `waulter.eu` zone.

Add two records:

| Type  | Name  | Value                              | TTL  |
|-------|-------|------------------------------------|------|
| CNAME | docs  | `<name>.azurestaticapps.net`       | 3600 |
| TXT   | docs  | `<azure-validation-token>`         | 3600 |

> **Wedos note**: Wedos propagates DNS changes within ~15 minutes for their
> own nameservers. Allow up to 1 hour for full propagation before Azure
> validates the domain ownership.

### Step 3 — Complete domain validation in Azure Portal

- Return to SWA → Custom domains → `docs.waulter.eu` → Validate
- Azure automatically provisions a **free managed TLS certificate** (Let's Encrypt)
  — no manual certificate management needed
- Once validated, `https://docs.waulter.eu` is live

### Step 4 — Update mkdocs.yml site_url

After the custom domain is confirmed:
```bash
# In mkdocs.yml, ensure:
site_url: https://docs.waulter.eu
```
Commit and push to trigger a redeploy.

## Custom domain: docs.stage.waulter.eu (Stage)

### Step 1 — Add custom domain in Azure Portal

- Stage SWA resource → Custom domains → Add
- Enter: `docs.stage.waulter.eu`
- Copy the CNAME target and validation token

### Step 2 — Create DNS records in Wedos CZ

Log in to [client.wedos.com](https://client.wedos.com) → DNS Manager →
select `waulter.eu` zone.

Add two records:

| Type  | Name       | Value                                    | TTL  |
|-------|------------|------------------------------------------|------|
| CNAME | docs.stage | `<stage-name>.azurestaticapps.net`       | 3600 |
| TXT   | docs.stage | `<azure-validation-token>`               | 3600 |

> **Note**: `docs.stage` creates the subdomain `docs.stage.waulter.eu`.

### Step 3 — Complete domain validation in Azure Portal

- Return to Stage SWA → Custom domains → `docs.stage.waulter.eu` → Validate
- Azure provisions a free managed TLS certificate automatically
- Once validated, `https://docs.stage.waulter.eu` is live

## Branch workflow

```
feature/* ──PR──▶ development ──PR──▶ main
                      │                  │
                   Stage SWA         Prod SWA
              docs.stage.waulter.eu  docs.waulter.eu
```

- **Feature branches** → open PRs against `development` → stage PR previews
- **development** → merges deploy to stage automatically
- **Promotion** → open PR from `development` to `main` → prod PR preview
- **main** → merges deploy to production automatically

- Custom domain + free TLS
- PR preview environments
- Sufficient for a documentation site

Two Free-tier SWA resources = no additional cost.
